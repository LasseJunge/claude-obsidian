# OpenClaw (openclaw.ai + docs, fetched 2026-06-24)

## What it is
Open-source, self-hosted "gateway that connects your favorite chat apps and channel surfaces to AI coding agents." Tagline: "The AI that actually does things." Runs locally (Mac/Windows/Linux). Creator: Peter Steinberger (@steipete). Repo: github.com/openclaw/openclaw. Built on Node.js (pnpm workspaces). Supports Claude, GPT, and other LLM backends.

## Architecture
A central **Gateway** process is the single source of truth for sessions, routing, and channel connections. Chat apps connect to the Gateway, which routes to agents, CLI, web UI, and mobile nodes.

## Features
Local execution (private by default), persistent memory, browser control, system access (files/shell/scripts with sandbox options), extensibility via community skills (ClawHub), multi-chat support (9+ platforms), autonomous background actions / cron.

## Channels
WhatsApp, Telegram, Discord, Slack, Signal, iMessage, Mattermost, Matrix, and more.

## Mattermost channel
Installs via `openclaw plugins install`; needs a bot token + base URL + DM policy. WebSocket events for channels/groups/DMs.
- Chat modes: oncall (@mention only, default), onmessage (all), onchar (prefix trigger like `>` / `!`).
- Threading via `replyToMode` (off / first / all).
- Access control: DM **pairing** policy by default (approval codes); channel allowlists by user ID or access group.
- Native slash commands: `oc_*` with HMAC-SHA256 token validation.
- Streaming: preview streaming consolidates thinking/tool activity/partial text into a single editable draft post.
- Reactions tool, interactive buttons (2D arrays, styles), directory adapter for `#channel`/`@user`, multi-account support.

## Security model (gateway/security)
**One trusted operator boundary per gateway (single-user, personal-assistant model).** Explicitly disclaims hostile multi-tenant isolation on shared infrastructure.
- Gateway requires auth by default (token/password). DM pairing codes expire after 1h; sessionKey is for context selection, NOT authorization.
- Sandboxing: tool exec in isolated Docker per agent; workspace access none/read-only/read-write; cross-agent access blocked by default.
- Permissions: root-bounded file access via `@openclaw/fs-safe`; state dir 700; credentials segregated.
- Privacy: transcripts on disk under `~/.openclaw/agents/` (filesystem access = trust boundary); log redaction on by default.
- NOT a security boundary against: prompt injection alone, a single shared gateway for adversarial users (split by OS user/host instead), session keys as per-user auth, node pairing as hidden approval.

## Streaming (concepts/streaming)
Block streaming (completed blocks) + preview streaming (Telegram/Discord/Slack/Matrix). Preview modes: off / partial / block / progress. Live status lines ("searching the web", "reading file") during tool execution.
